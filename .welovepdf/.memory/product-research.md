# WeLovePDF Product Memory

## Goal
Build a modular PDF tools ecosystem inspired by the market category led by iLovePDF, but with our own brand, user experience, architecture, and India-focused workflows.

## Research Notes
- iLovePDF groups tools into Organize PDF, Optimize PDF, Convert PDF, Edit PDF, PDF Security, and PDF Intelligence.
- High-demand public tools appear to be: JPG to PDF, PDF to Word, PDF to JPG, Merge PDF, Compress PDF, Split PDF.
- Their ecosystem includes web tools, premium accounts, business/team plans, API, desktop app, mobile app, workflows, and trust/security messaging.
- Their trust layer emphasizes encrypted processing, automatic file deletion, GDPR/security positioning, and business readiness.

## Our Initial Product Direction
- Start with a focused MVP instead of trying to launch 30+ tools at once.
- Prioritize India-heavy use cases: compress under upload limits, JPG to PDF for documents/forms, merge marksheets, split application PDFs, rotate scanned pages.
- Keep every tool isolated behind a shared job contract so future tools do not break existing ones.
- Public pages should be SEO-friendly and fast. Dashboard should be dense, calm, and operational.
